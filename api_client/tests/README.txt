В моей архитектуре взаимодействия с API есть 3 базовых класса: client, request, response

Они работают по следующей схеме(на примере VK API);
1) GUI вызывает метод getPlayLists() класса VKClient
2) Этот метод вызывает от указателя на класс VKRequest метод execute()
3) Он, в свою очередь, вызывает метод класса IRequestMaker, описанный в клиентской части сервера
4) После того, как сервер получил ответ от VK API, он вызывает метод get_response() класса 
VKResponse, чтобы передать ему ответ
5) Затем класс VKResponse обрабатывает ответ в методе parse_result() и вызывает метод 
GUI, через который он возвращает список запрошенных плейлистов

Таким образом, у нас получилась цепочка вызовов от одного класса методов другого.
И чтобы протестировать эту цепочку, я написал mock тесты для этих классов.
Написать тесты для конкретных случаев и значений на данном этапе разработки не представляется
возможным, ведь у нас нет полной реализации классов.
